# legged_gym 代码框架学习笔记

## 1. legged_gym 是什么

`legged_gym` 是一个腿式机器人强化学习训练框架，主要负责机器人仿真环境部分。

它负责：

* 创建机器人仿真环境
* 定义 observation
* 定义 action 如何执行
* 定义 reward 怎么计算
* 定义 reset 规则
* 定义 command 随机生成
* 加载机器人 URDF / mesh 身体模型
* 保存训练日志和模型 checkpoint

真正的 PPO 算法不在 `legged_gym` 里，而是在 `rsl_rl` 里。

一句话：

```text
legged_gym 负责机器人环境；
rsl_rl 负责 PPO 强化学习算法。
```

## 3. scripts：训练和播放入口

路径：

```text
legged_gym/scripts/
```

最重要两个文件：

```text
train.py
play.py
```

---

### 3.1 train.py

`train.py` 是训练入口。

常用命令：

```bash
python legged_gym/scripts/train.py \
  --task=anymal_c_flat \
  --headless \
  --num_envs=256 \
  --max_iterations=1000
```

它主要做三件事：

```text
1. 根据 task 创建机器人环境
2. 创建 PPO runner
3. 开始训练
```

核心逻辑：

```python
env, env_cfg = task_registry.make_env(...)
ppo_runner, train_cfg = task_registry.make_alg_runner(...)
ppo_runner.learn(...)
```

理解：

```text
train.py = 训练启动按钮
```

---

### 3.2 play.py

`play.py` 用来播放训练好的模型。

常用命令：

```bash
python legged_gym/scripts/play.py \
  --task=anymal_c_flat \
  --load_run Jun25_19-28-09_ \
  --checkpoint 1000 \
  --num_envs=16
```

它主要做：

```text
1. 创建同样的机器人环境
2. 加载 model_xxx.pt
3. 让机器人执行训练好的策略
4. 显示速度、关节、torque、接触力等图
```

理解：

```text
play.py = 模型效果查看器
```

## 6. envs/base：通用腿式机器人核心

路径：

```text

```

最重要两个文件：

```text
legged_robot.py
legged_robot_config.py
```

---

### 6.1 legged_robot.py

这是环境逻辑主文件，也是后面最值得反复看的文件。

里面有很多核心函数：

```text
step()
post_physics_step()
compute_observations()
compute_reward()
_prepare_reward_function()
_compute_torques()
check_termination()
reset_idx()
_resample_commands()
_reset_dofs()
_reset_root_states()
```

完整一步仿真大概是：

```text
PPO 给 action
↓
step(actions)
↓
_compute_torques()
↓
Isaac Gym simulate
↓
post_physics_step()
↓
check_termination()
↓
compute_reward()
↓
reset_idx()
↓
compute_observations()
↓
返回 obs / reward / done 给 PPO
```

理解：

```text
legged_robot.py = 机器人环境主逻辑
```

它决定：

```text
action 怎么执行
torque 怎么算
reward 怎么算
observation 怎么拼
机器人什么时候 reset
commands 什么时候重新随机
```

---

### 6.2 legged_robot_config.py

这是通用默认配置文件。

里面通常有这些配置类：

```text
env
terrain
commands
init_state
control
asset
rewards
normalization
noise
domain_rand
sim
```

它定义：

```text
num_envs
num_observations
地形类型
command 范围
初始姿态
PD 控制参数
reward 权重
噪声
domain randomization

```

例如 control 配置：

```python
class control:
    control_type = 'P'
    action_scale = 0.5
    stiffness = {...}
    damping = {...}
    decimation = 4
```

理解：

```text
legged_robot_config.py = 通用默认配置模板
```


### 7.1 AnymalCFlatCfg

这是环境配置。

它决定：

```text

地形是什么
```

例如：


```text

```


例如：



## 11. anymal_c_flat 的完整训练流程

你运行：

```bash
python legged_gym/scripts/train.py \
  --task=anymal_c_flat \
  --headless \
  --num_envs=256 \
  --max_iterations=1000
```

内部流程：

```text
train.py
↓
读取命令行参数 --task=anymal_c_flat
↓
import legged_gym.envs
↓
envs/__init__.py 注册所有任务
↓
task_registry.make_env("anymal_c_flat")
↓
找到 LeggedRobot + AnymalCFlatCfg
↓
创建 Isaac Gym 仿真环境
↓
task_registry.make_alg_runner("anymal_c_flat")
↓
找到 AnymalCFlatCfgPPO
↓
创建 rsl_rl 的 OnPolicyRunner
↓
runner.learn()
↓
PPO 开始训练
```

训练循环内部：

```text
observation
↓
actor 输出 action
↓
LeggedRobot.step(action)
↓
action 经过 PD 控制变成 torque
↓
Isaac Gym 推进仿真
↓
环境计算 reward
↓
环境生成新 observation
↓
PPO 存 rollout
↓
PPO update 更新 actor / critic
↓
保存 model_xxx.pt
```

## 12. 后面最常改的地方

### 12.1 改 reward 权重

主要文件：

```text
legged_gym/envs/anymal_c/flat/anymal_c_flat_config.py
```

重点位置：

```python
class rewards:
    class scales:
        torques = ...
        orientation = ...
        feet_air_time = ...
```

常见影响：

```text
torques 更负：更省力，但太大会不愿意动
orientation 更负：身体更稳，但太强可能不敢运动
feet_air_time 更大：更鼓励迈步，但太大可能乱跳
```

---

### 12.2 改 command 范围

通常也在 config 里：

```python
class commands:
    class ranges:
        lin_vel_x = [...]
        lin_vel_y = [...]
        ang_vel_yaw = [...]
```

作用：

```text
command 决定机器人训练时被要求做什么动作
```

例如只训练向前走：

```python
lin_vel_x = [0.5, 1.0]
lin_vel_y = [0.0, 0.0]
ang_vel_yaw = [0.0, 0.0]
```

---

### 12.3 改 action / control 参数

可能在：

```text
legged_gym/envs/base/legged_robot_config.py
```

或被具体机器人 config 覆盖。

重点：

```python
action_scale
stiffness
damping
decimation
```

作用：

```text
action_scale 决定 action 对目标关节角度影响多大
stiffness 是 PD 控制里的 P 增益
damping 是 PD 控制里的 D 增益
decimation 决定一个 action 执行多少个物理步
```

---

### 12.4 看 observation 怎么拼

这里能看到机器人到底观察了哪些信息。

### 12.5 看 reward 真正怎么算

文件：

```text

```

搜索：

```text

```

这里能看到每个 reward 的具体计算方式。

### 12.6 看 action 怎么变 torque

文件：


搜索：

## 13. 一句话总结整个框架

完整主线：

```text
train.py 按下训练按钮
↓
task_registry 根据 task 找环境和配置
↓
config 决定训练规则
↓
legged_robot.py 执行仿真、算 reward、生成 observation
↓
rsl_rl 的 PPO 根据 reward 更新策略
↓
logs 保存 model_xxx.pt
```

压缩理解：

```text
scripts 是入口；
utils 负责注册；
envs 是环境核心；
base 是通用机器人逻辑；
anymal_c/flat 是当前任务配置；
resources 是机器人身体；
logs 是训练出来的大脑。
```

最终记忆：

```text
legged_gym = 机器人训练环境
rsl_rl = PPO 算法
resources = 机器人身体
logs/model_xxx.pt = 训练出来的大脑
```
