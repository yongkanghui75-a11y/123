# legged_gym 跑通闭环流程笔记

## 1. 本阶段目标

这一阶段的目标不是改代码，而是先跑通一条完整训练闭环：

```text
安装环境
↓
训练 ANYmal C 平地任务
↓
保存模型
↓
用 TensorBoard 看训练曲线
↓
加载模型并播放
```

完整闭环是：

```text
train.py 训练
↓
logs/ 保存模型和日志
↓
TensorBoard 查看 reward 曲线
↓
play.py 加载模型
↓
可视化检查机器人是否能动
```

---

## 2. 进入环境

每次开始前先执行：

```bash
conda activate rlgpu
cd ~/rl_ws/legged_gym
```

这是进入强化学习训练环境和项目目录。

---

## 3. 小规模测试训练

第一次先跑小测试，确认环境能正常训练：

```bash
python legged_gym/scripts/train.py \
  --task=anymal_c_flat \
  --headless \
  --num_envs=64 \
  --max_iterations=10
```

含义：

```text
--task=anymal_c_flat
训练 ANYmal C 平地任务

--headless
不打开图形窗口，后台训练

--num_envs=64
同时训练 64 个机器人

--max_iterations=10
只训练 10 轮，用来测试流程
```

如果能跑到 `Learning iteration 9/10`，说明训练流程基本正常。

---

## 4. 正式 baseline 训练

小测试成功后，跑一个正式一点的 baseline：

```bash
python legged_gym/scripts/train.py \
  --task=anymal_c_flat \
  --headless \
  --num_envs=256 \
  --max_iterations=300
```

含义：

```text
同时训练 256 个机器人。
训练 300 轮。
这是一个短 baseline，不是最终好模型。
```

训练时重点看这些指标：

```text
Mean reward
平均奖励，整体越高越好。

Mean episode length
每轮机器人平均坚持多久。

rew_tracking_lin_vel
前进/横向速度跟踪奖励，越高越好。

rew_tracking_ang_vel
转向速度跟踪奖励，越高越好。

rew_orientation
身体姿态惩罚，越接近 0 越好。

rew_torques
关节用力惩罚，越接近 0 越省力。

rew_dof_acc
关节加速度惩罚，越接近 0 越不抖。
```

---

## 5. 查看模型保存位置

训练结束后查看模型文件：

```bash
find logs -name "*.pt" | tail
```

训练后的模型一般保存在：

```text
logs/flat_anymal_c/某个时间目录/model_XXX.pt
```

我之前跑出的模型路径类似：

```text
logs/flat_anymal_c/Jun20_14-37-00_/model_300.pt
```

含义：

```text
flat_anymal_c
实验名

Jun20_14-37-00_
某一次训练的 run 目录

model_300.pt
训练到第 300 轮保存的模型
```

---

## 6. 用 TensorBoard 查看训练曲线

启动 TensorBoard：

```bash
tensorboard --logdir /home/hyk/rl_ws/legged_gym/logs --port 6007 --load_fast=false
```

然后浏览器打开：

```text
http://localhost:6007
```

进入后选择：

```text
SCALARS
```

重点看：

```text
Episode/rew_tracking_lin_vel
机器人是否越来越能跟踪目标平移速度。

Episode/rew_tracking_ang_vel
机器人是否越来越能跟踪目标转向速度。

Episode/rew_orientation
身体是否越来越稳定。

Episode/rew_torques
动作是否越来越省力。

Episode/rew_dof_acc
动作是否减少抖动。

Loss/value_function
critic 价值函数损失。

Loss/surrogate
PPO 策略优化相关损失。
```

TensorBoard 的作用：

```text
终端只能看到当前训练状态；
TensorBoard 可以看到整个训练过程的曲线变化。
```

## 7. 播放训练好的模型

使用 `play.py` 加载模型：

```bash
python legged_gym/scripts/play.py \
  --task=anymal_c_flat \
  --load_run Jun20_14-37-00_ \
  --checkpoint 300 \
  --num_envs=16
```

含义：

```text
--task=anymal_c_flat
加载 ANYmal C 平地任务。

--load_run Jun20_14-37-00_
加载这一轮训练目录。

--checkpoint 300
加载 model_300.pt。

--num_envs=16
播放时只显示/运行 16 个机器人。
```

如果窗口里机器人能动起来，说明：

```text
训练 → 保存模型 → 加载模型 → 播放验证
```

这条闭环已经跑通。

