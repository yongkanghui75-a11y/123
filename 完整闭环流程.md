一、什么叫“跑通一个闭环训练”

在 legged_gym 里，一个完整闭环不是只运行训练命令，而是包括：

1. 进入正确环境
2. 启动训练
3. 训练生成日志和模型
4. 用 TensorBoard 看训练曲线
5. 用 play.py 加载模型并播放机器人
6. 判断训练效果

简单说就是：

训练 → 保存模型 → 看曲线 → 播放模型 → 判断效果

你昨天已经完成了这个闭环。

二、第一步：进入正确环境和项目目录

每次开始前先执行：

conda activate rlgpu
cd ~/rl_ws/legged_gym

确认终端左边是：

(rlgpu)

这一步很重要，因为你的 rlgpu 环境里才有：

Isaac Gym
源码编译版 PyTorch
rsl_rl
legged_gym
CUDA sm_120 支持

如果不小心在 (base) 里运行，可能会出错。

三、第二步：运行训练命令

你昨天跑的是：

python legged_gym/scripts/train.py \
  --task=anymal_c_flat \
  --headless \
  --num_envs=256 \
  --max_iterations=300

它的意思是：

python legged_gym/scripts/train.py
运行训练入口文件

--task=anymal_c_flat
训练 ANYmal C 四足机器人平地行走任务

--headless
不打开图形窗口，只后台训练，速度更快

--num_envs=256
同时模拟 256 个机器人并行训练

--max_iterations=300
训练 300 轮 PPO 迭代

运行后终端会出现：

Learning iteration 0/300
Learning iteration 1/300
...
Learning iteration 299/300

这说明训练正在进行。

四、第三步：观察训练终端输出

训练过程中重点看这些：

Mean reward
Mean episode length
rew_tracking_lin_vel
rew_tracking_ang_vel
rew_torques
rew_dof_acc
rew_orientation

大白话解释：

Mean reward
总体奖励，通常越高越好

Mean episode length
机器人平均能坚持多久，越长说明越不容易摔

rew_tracking_lin_vel
前进速度跟踪奖励，越高说明越会按目标速度走

rew_tracking_ang_vel
转向速度跟踪奖励，越高说明越会按目标方向转

rew_torques
关节用力惩罚，是负数，越接近 0 越省力

rew_dof_acc
关节加速度惩罚，是负数，越接近 0 越不抖

rew_orientation
身体姿态惩罚，是负数，越接近 0 越稳定

你昨天最后跑到：

Learning iteration 299/300
Total timesteps: 1843200
Total time: 94.23s
Mean reward: 0.18

说明训练正常完成。

五、第四步：确认模型保存位置

训练结束后执行：

cd ~/rl_ws/legged_gym
ls logs
find logs -name "*.pt" | tail

你昨天看到模型保存在：

logs/flat_anymal_c/Jun20_14-37-00_/model_300.pt

这里的含义是：

logs
所有训练日志和模型都在这里

flat_anymal_c
实验名称

Jun20_14-37-00_
这一次训练的时间文件夹

model_300.pt
第 300 轮保存的模型

.pt 文件就是训练好的策略网络参数。

六、第五步：打开 TensorBoard 看曲线

先启动 TensorBoard：

conda activate rlgpu
cd ~/rl_ws/legged_gym

tensorboard --logdir /home/hyk/rl_ws/legged_gym/logs --port 6007 --load_fast=false

然后浏览器打开：

http://localhost:6007

如果正常，会看到：

flat_anymal_c/Jun20_14-37-00_

选择 SCALARS 页面，主要看：

rew_tracking_lin_vel
rew_tracking_ang_vel
rew_torques
rew_dof_acc
rew_orientation

判断标准：

tracking_lin_vel 越高越好
tracking_ang_vel 越高越好
torques 越接近 0 越省力
dof_acc 越接近 0 越平滑
orientation 越接近 0 越稳定

你昨天的结论是：300 轮有改善，但改善有限。
这是正常的，因为 300 轮只是小 baseline，不是最终训练。

七、第六步：播放训练好的模型

训练完后，用 play.py 加载模型：

conda activate rlgpu
cd ~/rl_ws/legged_gym

python legged_gym/scripts/play.py \
  --task=anymal_c_flat \
  --load_run Jun20_14-37-00_ \
  --checkpoint 300 \
  --num_envs=16

这条命令的意思是：

play.py
播放训练好的模型

--task=anymal_c_flat
播放 ANYmal C 平地任务

--load_run Jun20_14-37-00_
加载这一次训练的日志文件夹

--checkpoint 300
加载 model_300.pt

--num_envs=16
显示 16 个机器人

如果窗口里能看到机器人运动，说明：

模型保存成功
模型加载成功
策略可以控制机器人
Isaac Gym 可视化正常

你昨天已经成功播放了机器人，所以闭环完成。

九、你以后每次实验都按这个模板

以后你改奖励、改环境数量、改训练轮数，都按这个流程：

1. 训练
2. 看终端输出
3. 找模型
4. 看 TensorBoard 曲线
5. play.py 播放
6. 记录效果

比如以后你可以做实验记录表：

实验	num_envs	max_iterations	改动	Mean reward	机器人表现
baseline	256	300	不改	0.18	能动，改善有限
longer_train	256	1000	不改	待记录	待观察
torque_test	256	300	改 torque 惩罚	待记录	待观察
